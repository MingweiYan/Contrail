# Contrail 

代码使用 Trae 来开发，非 Solo 模式，主要是 Doubao-seed-1.6 max 模式
icon 等图像资源使用豆包来生成

## Motivation 

为什么要做这个 App ?
- 主要看到很多大佬们分享的 vibe coding 相关的经验，感觉这是一件很酷的事情，想尝试一下
- 此前试过很多的类似的 App，大部分 App 需要考虑通用性，肯定不能百分百满足自己的要求，所以有自己定制化的想法

为什么 App 名字叫Contrail ？
    Contrail --- 凝结尾；航迹云；凝迹（飞机飞过留下的尾迹）。飞机飞过，会在天空上留下一道白线，我们的朝着目标的每次前进，也会留下痕迹，想做的就是记录下来自己朝着目标努力的痕迹。


核心的驱动力：
在《认知觉醒》中看到一个观点感觉非常触动，为了完成一个目标不要打卡，而是记录下自己的进展
> - 打卡会导致动机转移：一开始，人们的行动动机可能是出于学习成长本身，但打卡一旦开始，随着时间推移，热情消退，人们不得不依靠意志力坚持，此时大脑会开启自我保护模式，为了完成打卡任务而不自觉地降低标准，坚持的动机就会从学习本身转移到完成任务上，由内在需求转移到外在形式上。
> - 打卡会产生认知闭合效应：打卡活动自带任务心态，每次打卡都要面临一次任务闭合的需求。这会导致人们为了打卡而仓促结束学习，学习质量大打折扣，比如每天记 20 个单词，完成打卡后学习动机就衰减为零，不会再想着多做些探索。
>- 打卡会造成身心分裂：打卡造就了任务心态，这种心态会使注意力处于分散状态，很难全身心投入事物本身，从而体会其中的要领和乐趣，无法让人们享受过程，只会让身心紧张、焦虑、麻木和分裂，破坏了身心合一的状态。
>- 记录更关注学习本身：记录的核心是如实反映学习过程和状态，更关注学到了什么，而不是单纯标记是否完成，有助于培养沉浸和探索的习惯。同时，记录没有打卡的任务压力，可以将注意力集中到活动本身，让人们更专注于学习成长。


所以这个 App 的目标只是记录，不会提醒用户去完成打卡。只是提供一个方便的工具，记录行为并且统计结果，在用户需要的时候，可以满足用户对一定周期内的记录变化的分析


## 功能介绍

### 1. 习惯管理
- **习惯创建与编辑**：支持创建、编辑和删除习惯
- **习惯个性化**：每个习惯包含名称、图标和颜色，支持自定义
- **灵活追踪设置**：可选择追踪时间或仅追踪次数
- **目标设置**：支持设置正向目标（希望超过）或反向目标（希望低于）
- **多样化周期**：支持固定时间、每周、每月等多种周期设置

### 2. 习惯追踪（专注功能）
- **多种专注模式**：支持正计时、番茄钟、倒计时三种追踪模型
- **专注控制**：提供开始、暂停、恢复、结束功能
- **屏幕常亮**：支持专注时屏幕常亮，减少干扰
- **后台保护**：通过前台通知服务防止应用被系统杀死，保证计时准确性
- **补充记录**：支持手动补充历史专注记录

### 3. 习惯统计分析
- **多维度统计**：支持按周、月、年查看习惯完成情况
- **数据可视化**：提供折线图展示习惯完成趋势，饼状图展示各习惯在追踪次数和时间上的占比
- **日历与热力图**：支持月视图日历和年视图热力图，直观展示完成情况
- **完成率计算**：自动计算各习惯的完成率和平均连续天数

### 4. 其他特性
- **主题设置**：支持更换不同颜色的主题，不同的视觉体验
- **数据导出**：支持将数据导出为JSON格式，便于迁移和存档
