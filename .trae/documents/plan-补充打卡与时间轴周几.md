目标
- 补充打卡默认起始时间按“当前时间-时长”自动计算，且时长调整时起始时间同步更新
- 统计明细时间轴在日期下方展示星期信息，并兼容历史记录数据

范围确认
- 仅涉及补充打卡弹窗逻辑与统计明细时间轴 UI 展示
- 不变更数据结构与数据库存储格式，兼容已有 trackingDurations 数据

实施步骤
1. 分支与现状确认
   - 创建新分支（命名遵循现有约定）
   - 确认补充打卡弹窗的默认起始时间、时长滑杆逻辑与统计明细时间轴展示位置

2. 补充打卡默认起始时间与联动
   - 在补充打卡弹窗初始化时，固定“默认结束时间 = 当前时间”
   - 基于“结束时间 - 时长”计算默认起始日期/时间，填充 selectedDate/selectedTime
   - 当时长滑杆变更时，重新计算起始日期/时间并更新 UI
   - 确保 trackTime=false 场景不受影响（仍可补充记录）

3. 时间轴日期下方显示星期
   - 在时间轴卡片中添加“星期显示”区域，位于日期下方
   - 星期通过 startTime 的日期计算（不依赖新增字段），确保历史数据可展示
   - 处理可能的空值或异常数据，保证 UI 稳定

4. 验证
   - 手工验证：补充打卡弹窗默认时间是否为“当前时间-时长”，修改时长是否同步变化
   - 手工验证：统计明细时间轴每条记录日期下方是否显示星期，且历史数据可正常展示

输出
- 更新补充打卡弹窗的默认时间与联动逻辑
- 统计明细时间轴新增星期显示
