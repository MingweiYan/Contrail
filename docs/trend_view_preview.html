趋势视图 UI 预览

这次只做预览，不改真实 Flutter 代码。目标是优化统计页中的趋势区域，以及习惯明细统计页里的趋势模块，让控制区更集中、图表更有主次、单习惯趋势更容易读。

优化方向

重点在结构、层级和图表承载方式，不是单纯换颜色
方案 A

控制台总览式

把时间切换、周期切换、摘要指标和总图合并成一个大面板，适合统计总览页的趋势模式。优点是进入后信息完整，不用来回扫多个卡片。

趋势模式

习惯趋势总览

把时间范围、周期筛选和趋势摘要放进同一个头部面板，降低当前统计页里卡片碎片化的感觉。

方案 B

双列分析式

左边放主图，右边放当前周期的关键说明和习惯排行，强调“读图 + 解读”。适合统计页趋势区，尤其在多习惯时更容易建立比较关系。

完成率趋势分析

主图负责走势，右侧说明负责“结论”，减少用户只看到线条、看不出重点的问题。

方案 C

单习惯聚焦式

给习惯明细统计页用。把当前习惯的趋势图做成主卡，上面是时间范围和周期切换，下面是单独的次数/时间双轨图，减少“通用统计组件搬过来”的割裂感。

方案 D

时间轴对照式

还是给习惯明细趋势页用，但更偏“回顾感”。把图表和阶段结论做成竖向时间轴式分段，更适合周期复盘和个人习惯演化查看。

专注轨迹回顾

强调“这段时间发生了什么”，而不是只给一张折线图。适合单习惯详情页做更强的故事感。
